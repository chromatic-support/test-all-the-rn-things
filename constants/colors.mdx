## Dark Theme

Dark mode colors are defined in `Colors.dark` in [theme.ts](./theme.ts). Use the `Colors` export with React Native's `useColorScheme` hook to switch between light and dark tokens automatically.

```tsx
import { useColorScheme } from 'react-native';
import { Colors } from '@/constants/theme';

const colorScheme = useColorScheme();
const colors = Colors[colorScheme ?? 'light'];
```

## Usage

Colors are exported from [theme.ts](./theme.ts) as `Palette` (raw values) and `Colors` (semantic light/dark tokens).

### Basic Usage

```tsx
import { Palette, Colors } from '@/constants/theme';

// Raw palette value
<View style={{ backgroundColor: Palette.peach }} />

// Semantic token (respects light/dark mode)
<Text style={{ color: colors.text }} />
```

### Using Grays

```tsx
<View style={{
  backgroundColor: Palette.gray100,
  borderColor: Palette.gray300,
}}>
  <Text style={{ color: Palette.gray900 }}>Card content</Text>
</View>
```

### Using Base Colors

```tsx
<View style={{ backgroundColor: Palette.mint }}>
  <Text style={{ color: Palette.black }}>Accent box</Text>
</View>
```

### Using Color Variants for Depth

Each base color has light and dark variants for creating visual depth and interactive states.

```tsx
// Button using color variants
<Pressable style={({ pressed }) => ({
  backgroundColor: pressed ? Palette.mintDark : Palette.mint,
  borderColor: Palette.mintDark,
  borderWidth: 2,
})} />
```

## Best Practices

1. **Prefer semantic tokens** (`Colors.light` / `Colors.dark`) over raw `Palette` values for text, backgrounds, and icons
2. **Use raw palette values** for decorative elements and accents where light/dark distinction isn't needed
3. **Test dark mode** by toggling the device/simulator appearance
4. **Ensure sufficient contrast** for accessibility
5. **Use grays** for neutral UI elements
6. **Reserve accent colors** for important actions
