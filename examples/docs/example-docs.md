# DOCS EXAMPLES

## DO

- Use uppercase for `README.md` headers (e.g., `## USAGE`, not `## Usage`).
- Keep documentation close to the code it describes.
- Use markdown link syntax for referencing files and external resources.
- Add a general tips markdown file (`example-{name}.md`) to each `examples/` subdirectory.

## DO NOT

- Write documentation that duplicates what is already clear from the code.
- Use HTML tags in markdown files where plain markdown suffices.
- Leave subdirectories under `examples/` without a corresponding `example-{name}.md` file.
- Add docs for hypothetical or future functionality — document what exists today.
